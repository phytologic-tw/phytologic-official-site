## 補充二、植本誌（覆寫第5節，新增正式內容模型）

### 前台頁面結構

```
[輪播區塊]
  來源邏輯：優先顯示手動標記 featured 的文章；
            featured 數量不足時，自動以最新發布文章補滿輪播位
  呈現內容：海報圖、主題標題

[內容列表]
  排序：純依發布日期，新到舊，不做分類篩選
  每筆呈現：類別｜主題｜發布日期｜「前往觀看」連結
  類別：影片／文章／活動花絮
```

### CMS 內容模型欄位定義（草案，待 Sanity 實作時對照調整）

| 欄位 | 型別 | 說明 |
|---|---|---|
| `title` | 文字 | 標題 |
| `category` | enum | 影片／文章／活動花絮 |
| `publishedDate` | 日期 | 發布日期 |
| `featured` | 布林值 | 是否手動標記上輪播 |
| `posterImage` | 圖片 | 海報圖，輪播用 |
| `body` | 依類型而定 | 內容本體，可能是影片連結或圖文內容 |
| `requiresReview` | 布林值 | 是否需審核——健康文章類固定為 true，其餘類型可彈性 |
| `reviewStatus` | enum | 草稿／待審／已發布，串接 Sanity 原生工作流程 |

### 審核流程

健康文章（category = 文章 且內容涉及健康主張時）必須完整套用 V2 第5.0節內容邊界規則，經審核後才能發布。審核權限可指定角色，不寫死單一審核人，Bryan 為最高權限，可自行審核或授權他人。

### 技術選型

採用 Sanity（已於 `docs/JOURNAL_CMS_TECH_COMPARISON.md` 完成比較與推薦理由：開發時間短、審核工作流程原生支援、Free Tier 初期免費）。獨立資料庫，**不共用 PhytoWorld 會員系統的 Supabase 專案**（沿用 V2 規範）。

### 動態呈現規則

依 V3 第0節，植本誌**不強制**進場動畫（屬列表型持續更新內容）。
